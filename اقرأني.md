# 🎓 نظام جداول الكلية التقنية

## ✅ تم إعداد النظام بنجاح!

النظام الآن مرتبط بقاعدة بيانات PostgreSQL المحلية.

---

## 🚀 طريقة التشغيل السريعة

### الطريقة الأولى: استخدام الملفات المُعدة مسبقاً

1. **تشغيل السيرفر:**
   - انقر مرتين على ملف `start-server.bat`
   - انتظر حتى ترى رسالة "Connected to PostgreSQL database"

2. **تشغيل الواجهة:**
   - انقر مرتين على ملف `start-frontend.bat`
   - سيفتح المتصفح تلقائياً

### الطريقة الثانية: استخدام Terminal

#### تشغيل السيرفر:
```powershell
cd server
node index.js
```

#### في terminal جديد - تشغيل الواجهة:
```powershell
npm run dev
```

---

## 📋 معلومات قاعدة البيانات

- **نوع القاعدة:** PostgreSQL
- **اسم القاعدة:** `college_scheduler`
- **المستخدم:** `postgres`
- **كلمة المرور:** `admin123`
- **المنفذ:** `5432`

---

## 🌐 عناوين الوصول

- **الواجهة الأمامية:** http://localhost:5173
- **السيرفر (API):** http://localhost:3001
- **عبر الشبكة المحلية:** استخدم IP جهازك بدلاً من localhost

---

## 📊 هيكل قاعدة البيانات

النظام يُنشئ الجداول التالية تلقائياً:

| الجدول | الوصف |
|--------|-------|
| `exams` | معلومات الامتحانات (الكود، الاسم، التاريخ، الوقت، التخصص، القسم) |
| `rooms` | القاعات وسعتها |
| `proctors` | المراقبين وأقسامهم |
| `students` | الطلاب وتخصصاتهم |
| `student_courses` | ربط الطلاب بالمقررات |
| `committees` | اللجان الامتحانية |
| `committee_proctors` | توزيع المراقبين على اللجان |
| `committee_students` | توزيع الطلاب على اللجان |

---

## ⚙️ تغيير إعدادات قاعدة البيانات

إذا كنت تريد تغيير معلومات الاتصال بقاعدة البيانات:

1. افتح ملف `server/index.js`
2. ابحث عن هذا الجزء:

```javascript
const pool = new Pool({
    user: 'postgres',
    host: 'localhost',
    database: 'college_scheduler',
    password: 'admin123',
    port: 5432,
});
```

3. غيّر القيم حسب إعداداتك
4. احفظ الملف وأعد تشغيل السيرفر

## 🔧 استكشاف الأخطاء

### ❌ خطأ: Cannot connect to PostgreSQL

**الحل:**
- تأكد من أن PostgreSQL يعمل
- تحقق من وجود قاعدة البيانات `college_scheduler`
- تأكد من صحة كلمة المرور

**إنشاء قاعدة البيانات:**
```sql
CREATE DATABASE college_scheduler;
```

### ❌ خطأ: Port 3001 already in use

**الحل:**
- أوقف أي برنامج يستخدم المنفذ 3001
- أو غيّر المنفذ في `server/index.js`
