# نظام جداول الكلية التقنية - دليل التشغيل

## المتطلبات الأساسية

1. **PostgreSQL** - يجب أن يكون مثبتاً ويعمل على المنفذ 5432
2. **Node.js** - إصدار 14 أو أحدث
3. قاعدة بيانات باسم `college_scheduler`

## إعداد قاعدة البيانات

تأكد من أن قاعدة البيانات `college_scheduler` موجودة في PostgreSQL مع المعلومات التالية:
- اسم المستخدم: `postgres`
- كلمة المرور: `admin123`
- المنفذ: `5432`
- المضيف: `localhost`

## خطوات التشغيل

### 1. تشغيل الخادم (Backend)

افتح Terminal وقم بتنفيذ الأوامر التالية:

```powershell
cd "c:\Users\ADMIN\Desktop\last-version-of-نظام-جداول-الكلية-التقنية\server"
npm start
```

سيعمل الخادم على المنفذ `3001`

### 2. تشغيل الواجهة الأمامية (Frontend)

افتح Terminal جديد وقم بتنفيذ الأوامر التالية:

```powershell
cd "c:\Users\ADMIN\Desktop\last-version-of-نظام-جداول-الكلية-التقنية"
npm install
npm run dev
```

ستعمل الواجهة على المنفذ `5173` (أو حسب إعدادات Vite)

### 3. الوصول إلى النظام

افتح المتصفح وانتقل إلى:
```
http://localhost:5173
```

## هيكل قاعدة البيانات

يقوم النظام تلقائياً بإنشاء الجداول التالية:

- **exams** - معلومات الامتحانات
- **rooms** - القاعات
- **proctors** - المراقبين
- **students** - الطلاب
- **student_courses** - ربط الطلاب بالمقررات
- **committees** - اللجان
- **committee_proctors** - ربط اللجان بالمراقبين
- **committee_students** - ربط اللجان بالطلاب

## استكشاف الأخطاء

### خطأ في الاتصال بقاعدة البيانات

إذا ظهرت رسالة خطأ عند بدء التشغيل:
1. تأكد من أن PostgreSQL يعمل
2. تحقق من أن قاعدة البيانات `college_scheduler` موجودة
3. تأكد من صحة اسم المستخدم وكلمة المرور في ملف `server/index.js`

### تغيير معلومات الاتصال بقاعدة البيانات

لتغيير معلومات الاتصال، افتح ملف `server/index.js` وعدل هذا الجزء:

```javascript
const pool = new Pool({
    user: 'postgres',           // اسم المستخدم
    host: 'localhost',          // المضيف
    database: 'college_scheduler', // اسم قاعدة البيانات
    password: 'admin123',       // كلمة المرور
    port: 5432,                 // المنفذ
});
```

## ملاحظات مهمة

- النظام يقوم تلقائياً بإنشاء الجداول عند التشغيل لأول مرة
- جميع البيانات تُحفظ في قاعدة بيانات PostgreSQL
- تأكد من تشغيل الخادم قبل الواجهة الأمامية
- لإيقاف التشغيل، اضغط `Ctrl+C` في Terminal

## دعم الشبكة المحلية

السيرفر يعمل على `0.0.0.0` مما يسمح بالوصول من أجهزة أخرى على نفس الشبكة المحلية باستخدام IP الجهاز بدلاً من localhost.
