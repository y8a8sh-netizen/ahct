# 🔐 دليل الصلاحيات ونظام المصادقة

## نظرة عامة

تم تحديث النظام ليتضمن نظام مصادقة وصلاحيات متقدم يفصل بين المستخدمين حسب الأدوار.

---

## 📊 جدول الصلاحيات

| الدور | اسم المستخدم | كلمة المرور | الصلاحيات | الوصول |
|-------|--------------|-------------|-----------|--------|
| **مدير النظام** | `postgres` | `admin123` | ✅ قراءة وكتابة كاملة | كل الصفحات |
| **رئيس القسم** | - | `DepAdmin` | 📖 قراءة فقط | الكشوف والطباعة |
| **المراقب** | - | - | 📖 قراءة فقط | جدول المراقبة |
| **الطالب** | - | - | 📖 قراءة فقط | جدول الاختبارات |

---

## 🔑 تفاصيل الأدوار

### 1. مدير النظام (System Admin)

**بيانات الدخول:**
- اسم المستخدم: `postgres`
- كلمة المرور: `admin123`

**الصلاحيات:**
- ✅ رفع وتحديث البيانات (CSV)
- ✅ إضافة وتعديل القاعات
- ✅ إضافة وتعديل المراقبين
- ✅ إنشاء وتوزيع اللجان
- ✅ تعديل اللجان الموجودة
- ✅ حذف البيانات
- ✅ تصفير النظام
- ✅ تصدير التقارير
- ✅ الكتابة في قاعدة البيانات PostgreSQL

**ملاحظة:** هذا المستخدم الوحيد القادر على إجراء تغييرات في قاعدة البيانات.

---

### 2. رئيس القسم (Department Head)

**بيانات الدخول:**
- كلمة المرور: `DepAdmin`

**الصلاحيات:**
- 📖 عرض جميع البيانات
- 📖 طباعة الكشوف
- 📖 عرض اللجان والجداول
- 📄 تصدير التقارير (للقراءة فقط)
- ❌ لا يمكنه التعديل أو الحذف

---

### 3. المراقب (Proctor)

**بيانات الدخول:**
- دخول مباشر (بدون كلمة مرور)

**الصلاحيات:**
- 📖 عرض جدول المراقبة الخاص به
- 📖 معرفة القاعات والأوقات
- 📖 عرض أسماء الطلاب في لجنته
- ❌ لا يمكنه التعديل

---

### 4. الطالب (Student)

**بيانات الدخول:**
- دخول مباشر (بدون كلمة مرور)

**الصلاحيات:**
- 📖 عرض جدول الاختبارات
- 📖 معرفة القاعات والأوقات
- 📖 البحث عن اختبار معين
- ❌ لا يمكنه التعديل

---

## 🛡️ الحماية المطبقة

### على مستوى الواجهة (Frontend)

1. **تعطيل الأزرار:**
   - جميع أزرار الرفع معطلة
   - أزرار التعديل والحذف معطلة
   - زر إنشاء اللجان معطل
   - زر تصفير النظام معطل

2. **رسائل تحذيرية:**
   - بانر أصفر يظهر في أعلى الصفحة
   - رسائل عند تمرير الماوس على الأزرار المعطلة

3. **منع التفاعل:**
   - حقول الإدخال معطلة
   - النماذج غير قابلة للإرسال

### على مستوى قاعدة البيانات (Backend)

✅ **التوصية:** يجب إضافة فحص إضافي على مستوى السيرفر لمنع أي محاولات غير مصرح بها.

## 📝 سيناريوهات الاستخدام

### السيناريو 1: بداية الفصل الدراسي
```
1. مدير النظام يدخل بـ (postgres / admin123)
2. يرفع ملفات CSV للطلاب والاختبارات
3. يضيف القاعات والمراقبين
4. يقوم بإنشاء اللجان
5. يصدر الجداول النهائية
```

### السيناريو 2: مراجعة رئيس القسم
```
1. رئيس القسم يدخل بكلمة المرور (DepAdmin)
2. يستعرض اللجان والجداول
3. يطبع الكشوف
4. لا يمكنه إجراء أي تعديل
```

### السيناريو 3: اطلاع المراقب
```
1. المراقب يدخل مباشرة (بدون كلمة مرور)
2. يبحث عن اسمه
3. يرى جدول مراقبته (القاعات + الأوقات)
4. يطبع جدوله الشخصي
```

### السيناريو 4: اطلاع الطالب
```
1. الطالب يدخل مباشرة (بدون كلمة مرور)
2. يبحث عن رقمه التدريبي
3. يرى جدول اختباراته ومواقعها
4. يطبع جدوله الشخصي
```

## ⚠️ تحذيرات أمنية

### 🔴 مهم جداً:

1. **لا تشارك بيانات مدير النظام**
   - اسم المستخدم وكلمة المرور سرية
   - شاركها فقط مع الأشخاص المصرح لهم

2. **تغيير كلمة المرور**
   - يمكنك تغيير كلمة المرور من ملف `pages/LoginPage.tsx`
   - ابحث عن السطر: `inputUsername === 'postgres' && inputPassword === 'admin123'`

3. **حماية قاعدة البيانات**
   - غيّر كلمة مرور PostgreSQL الافتراضية
   - لا تعرض المنفذ 5432 للإنترنت مباشرة
